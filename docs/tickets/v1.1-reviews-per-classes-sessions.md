# Anonymous Class Reviews Feature

## Context

Students can anonymously rate each class session (0-5 stars + optional comment). Reviews are per session (identified by date), so if a class meets Mon+Wed, a student can review each session separately. Teachers see anonymous reviews but never know who wrote them. UI lives on the student's enrollments page. Tests cover both backend and frontend.

## Phase 1: Backend

### 1. Migration — `backend/database/migrations/1775800000000_create_reviews_table.ts`

- `id` UUID PK (`gen_random_uuid()`)
- `class_id` UUID FK → classes(id) CASCADE
- `student_id` UUID FK → users(id) CASCADE
- `rating` integer NOT NULL, CHECK 0-5
- `comment` text nullable
- `session_date` date NOT NULL — identifies which class session is being reviewed
- `created_at`, `updated_at`
- UNIQUE constraint on `(class_id, student_id, session_date)` — one review per student per session

### 2. Model — `backend/app/models/review.ts`

- Columns: id, classId, studentId, rating, comment, sessionDate, createdAt, updatedAt
- `@belongsTo(() => Class)`, `@belongsTo(() => User, { foreignKey: 'studentId' })`
- Update `backend/app/models/class.ts` to add `@hasMany(() => Review)`

### 3. Validator — `backend/app/validators/review_validator.ts`

- `createReviewValidator`: rating (number 0-5), comment (string, max 1000, optional), session_date (date string)
- `updateReviewValidator`: rating + comment (session_date not updatable)

### 4. Policy — `backend/app/policies/review_policy.ts`

- `create(user, cls)` — student + active enrollment
- `update(user, review)` — student + owns review
- `listForTeacher(user, cls)` — teacher + owns class

### 5. Controller — `backend/app/controllers/reviews_controller.ts`

| Method      | Route                                   | Who     | Notes                                                      |
| ----------- | --------------------------------------- | ------- | ---------------------------------------------------------- |
| `store`     | POST `/classes/:classId/reviews`        | student | body includes session_date; 409 if duplicate for that date |
| `update`    | PUT `/classes/:classId/reviews/:id`     | student | own review only                                            |
| `index`     | GET `/classes/:classId/reviews`         | teacher | anonymous (no student_id), supports ?session_date filter   |
| `summary`   | GET `/classes/:classId/reviews/summary` | teacher | avg + count, optionally filtered by session_date           |
| `myReviews` | GET `/my-reviews/:classId`              | student | all own reviews for a class                                |

**Anonymity guarantee**: index/summary responses NEVER include `student_id`.

### 6. Routes — modify `backend/start/routes.ts`

- Add review routes in the classes group + `my-reviews` in authenticated group

### 7. Backend Tests — `backend/tests/functional/reviews.spec.ts`

- Create: happy path, not enrolled (403), teacher (403), duplicate same date (409), different date OK, validation (422)
- Update: happy path, wrong student (403), not found (404)
- Index: teacher sees anonymous reviews, non-owner (403), student (403), **no student_id in response**
- Summary: correct avg/count
- My Reviews: returns own reviews, teacher (403)

## Phase 2: Frontend

### 8. Domain — `frontend/src/domains/reviews/`

- **types/review.types.ts**: Review (with session_date), ReviewSummary, CreateReviewInput, UpdateReviewInput
- **schemas/review.schema.ts**: Zod — rating (0-5), comment (max 1000, optional), session_date (date string)
- **services/reviews.service.ts**: createReview, updateReview, getClassReviews, getClassReviewSummary, getMyReviews
- **hooks/**: useMyReviews, useCreateReview, useUpdateReview, useClassReviews, useClassReviewSummary

### 9. Components

- **StarRating.tsx** — reusable 5-star interactive/readonly component
- **ReviewForm.tsx** — RHF + Zod, star rating + optional comment + session date picker, create/edit modes
- **ReviewSection.tsx** — goes inside each enrollment card; shows recent reviews + "Rate today's class" button
- **ReviewSummary.tsx** — teacher view: avg stars + count + anonymous review list

### 10. Integration

- **`frontend/src/routes/_authenticated/enrollments.tsx`** — add `<ReviewSection>` in each enrollment card
- **Teacher class detail** — add Reviews tab showing `<ReviewSummary>`

### 11. Frontend Tests

- StarRating: renders stars, click changes value, readonly mode
- ReviewForm: submit calls mutation, validation, edit mode pre-fills
- ReviewSection: shows rate button, shows existing reviews
