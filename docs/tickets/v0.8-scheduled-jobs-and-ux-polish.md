# v0.8 — Scheduled Jobs, Security Headers & UX Polish

## Main Goal

Complete all backend automation (scheduled jobs for cleanup and payment reminders), harden the API with security headers, and bring the frontend to a production-quality UX level: dashboards for both roles, empty states, loading states, error boundaries, responsive layout, and toast notifications. After this ticket the app is functionally complete and polished.

---

## Scope

### Backend

#### Scheduled Jobs (`@adonisjs/scheduler`)

**`CleanupExpiredInvitationsJob`** — runs daily at 02:00:

- Delete invitation rows where `expires_at < now()`.
- Log count of deleted rows.

**`PurgeExpiredNotificationsJob`** — runs daily at 02:30:

- Delete notification rows where `expires_at < now()`.
- Log count of deleted rows.

**`PurgeExpiredTokensJob`** — runs daily at 03:00:

- Delete revoked/expired auth tokens from the tokens table (Lucid auth token cleanup).

**`PaymentReminderJob`** — runs on a configurable schedule (default: monthly, 1st of each month at 09:00):

- For each active enrollment: if the class has a payment reminder configured (future feature hook — for now just log that reminder would be sent).
- Send email via `@adonisjs/mail` to the student with a "friendly reminder" template.
- This job is a stub that can be extended — the mailer is wired and a basic template exists.

**`@adonisjs/mail` configuration:**

- SMTP or Resend driver, configured via `MAIL_DRIVER`, `RESEND_API_KEY` / SMTP env vars.
- `MAIL_FROM` env var.
- Basic email template for payment reminder.

#### Security Headers Middleware

Custom `SecurityHeadersMiddleware` applied globally to all API responses:

- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `Strict-Transport-Security: max-age=31536000; includeSubDomains` (only in production)
- `X-XSS-Protection: 1; mode=block`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Permissions-Policy: camera=(), microphone=(), geolocation=()`

#### Backend Polish

- Ensure all error responses follow consistent shape: `{ error: { message: string, code?: string, fields?: Record<string, string[]> } }`
- Add `X-Request-ID` header to all responses (generated per request, logged for traceability).
- Pagination on list endpoints (cursor or offset): `GET /classes`, `GET /notifications`, `GET /feedback`, `GET /announcements`. Default page size: 20. Response includes `{ data: [...], meta: { total, page, per_page } }`.

### Frontend

#### Dashboards

**Teacher Dashboard (`/dashboard` when `profile_type === 'teacher'`)**:

- Summary cards:
  - Total students (across all classes)
  - Classes today (schedules matching current day_of_week)
  - Feedback sent this week
  - Unread notifications count
- Recent activity feed: last 5 notifications
- Quick links: "View my classes", "Create new class"

**Student Dashboard (`/dashboard` when `profile_type === 'student'`)**:

- Summary cards:
  - Classes enrolled in
  - Next upcoming class (closest schedule)
  - Recent belt (most recent BeltProgress)
  - Unread notifications
- Recent announcements feed (last 3 across all enrollments)
- Quick links: "View my classes", "My feedback", "Privacy center"

#### UX Polish

**Loading states:**

- All data-fetching components use a `Spinner` while `isLoading` is true.
- Skeleton loaders for card grids (ClassCard grid, AnnouncementFeed).

**Empty states:**

- "No classes yet. Create your first class." (teacher `/classes` with zero classes)
- "You're not enrolled in any classes yet." (student `/enrollments` with zero)
- "No announcements yet." (AnnouncementFeed)
- "No feedback yet." (FeedbackHistory)
- "No notifications." (NotificationPanel)
- Each empty state includes a relevant icon and a call-to-action where applicable.

**Error handling:**

- `ErrorBoundary` component wrapping each route: catches unexpected render errors, shows "Something went wrong" with a retry button.
- API errors (4xx, 5xx) surface as toast notifications using a `Toast` component (positioned top-right, auto-dismiss in 5s).
- Network error (fetch failed): shows toast "Connection error. Please check your internet."

**Responsive layout:**

- `Sidebar.tsx` collapses to `MobileNav.tsx` (bottom tab bar) on screens < 768px.
- All forms are usable on mobile (touch-friendly inputs, large tap targets).
- Card grids: 1 column on mobile, 2 on tablet, 3 on desktop.

**Design system completions (all in `shared/components/ui/`):**

- `Button.tsx` — variants: `primary`, `secondary`, `danger`, `ghost`. Sizes: `sm`, `md`, `lg`. Supports `isLoading` prop (shows spinner, disables click).
- `Input.tsx` — with label, error message, and optional helper text.
- `Modal.tsx` — accessible (focus trap, ESC to close, backdrop click to close).
- `Card.tsx` — consistent padding and shadow.
- `Badge.tsx` — for belt levels, enrollment status.
- `EmptyState.tsx` — icon + message + optional CTA button.
- `Spinner.tsx` — used across loading states.
- `Toast.tsx` — global toast container (single instance in root layout).

## Acceptance Criteria

- [ ] `CleanupExpiredInvitationsJob` deletes invitations where `expires_at < now()`. Running it manually in a test removes only expired invitations, not active ones.
- [ ] `PurgeExpiredNotificationsJob` deletes notifications where `expires_at < now()`.
- [ ] All API responses include security headers: `X-Content-Type-Options`, `X-Frame-Options`, `X-XSS-Protection`, `Referrer-Policy`.
- [ ] All API responses include `X-Request-ID` header with a unique value per request.
- [ ] List endpoints support pagination: `?page=1&per_page=20` query params. Response includes `meta.total`.
- [ ] Mailer is configured and can send a test email without throwing an error.
- [ ] Frontend dashboards show correct counts (connected to real API data, not hardcoded).
- [ ] Teacher dashboard "Classes today" card reflects the current day's schedule.
- [ ] All data-fetching routes show a spinner or skeleton while loading.
- [ ] All empty states are visible when the relevant data is absent.
- [ ] `ErrorBoundary` catches a simulated render error and shows the fallback UI.
- [ ] API 4xx/5xx errors display a toast notification with the error message.
- [ ] `Modal` traps focus while open (tabbing does not leave the modal). ESC key closes it.
- [ ] `Button` with `isLoading=true` shows a spinner and ignores click events.
- [ ] Layout is usable on a 375px-wide screen (mobile). No horizontal scroll.

## Tests

- **Cleanup job — expired invitations:** Insert 3 expired + 2 active invitations. Run job. Assert 3 deleted, 2 remain.
- **Cleanup job — expired notifications:** Insert 5 expired + 3 active notifications. Run job. Assert 5 deleted, 3 remain.
- **Security headers:** `GET /health` response includes `X-Content-Type-Options: nosniff` and `X-Frame-Options: DENY`.
- **X-Request-ID:** Two requests return different `X-Request-ID` values.
- **Pagination:** `GET /api/v1/classes?page=1&per_page=2` with 5 classes returns exactly 2 items and `meta.total = 5`.
- **Mailer — no error:** Calling `PaymentReminderJob` with a mock mail transport does not throw.

### Frontend (Vitest)

- **Button — isLoading:** When `isLoading=true`, clicking the button does not fire `onClick`. Spinner is rendered.
- **EmptyState:** Renders icon, message, and CTA button when provided. Snapshot test.
- **Modal:** On open, first focusable element receives focus. ESC key press fires `onClose`. Clicking backdrop fires `onClose`.
- **ErrorBoundary:** Wrapping a component that throws renders the fallback UI instead of propagating the error.
- **Toast:** Calling `showToast('message', 'error')` renders a toast. After 5000ms, it auto-dismisses (use fake timers).

- **Empty state — new teacher:** Register new teacher → navigate to `/classes` → see empty state with "Create your first class" CTA.
- **Mobile layout:** Set viewport to 375×812 → navigate app → no horizontal scroll → bottom nav visible.
