# v0.1 — Foundation & Infrastructure

## Main Goal

Establish the complete project scaffold: Docker services running, backend and frontend booting, database connected, migrations applied, and the encryption layer configured. No features yet — just a fully wired skeleton that every future ticket builds on.

---

## Scope

### Backend (AdonisJS 6)

- Scaffold AdonisJS 6 project inside `backend/`
- Configure PostgreSQL driver (`@adonisjs/lucid`) and [Valkey](https://glide.valkey.io/getting-started/quickstart/?lang=node) (high-performance key/value datastore that supports a variety of workloads such as caching, message queues) connection
- Configure `@adonisjs/encryption` with `APP_KEY`
- Write and run migrations:
  - `users` — `id` (UUID PK), `first_name`, `last_name` (enc), `email` (enc), `password_hash`, `birth_date` (enc), `profile_type` (`teacher` | `student`), `avatar_url`, `created_at`, `updated_at`, `deleted_at` (soft delete)
  - `student_profiles` — `id`, `user_id` (FK), `weight_kg` (enc, nullable), `height_cm` (enc, nullable), `data_consent_at`
  - `teacher_profiles` — `id`, `user_id` (FK), `fight_experience` (nullable)
  - `audit_logs` — `id`, `user_id`, `action`, `resource_type`, `resource_id`, `ip_address`, `metadata` (JSON), `created_at`
- Set up `@adonisjs/cors` (strict: only `VITE_APP_URL` origin allowed, no wildcard)
- Set up `@adonisjs/limiter` (wired, not yet applied to routes)
- Global exception handler that returns consistent JSON errors and never leaks stack traces, DB details, or internal paths
- Health check endpoint: `GET /health` → `{ status: 'ok' }`

### Frontend (Vite + TanStack Router SPA)

- Scaffold Vite + React + TypeScript project inside `frontend/`
- Install and configure TanStack Router (SPA mode, file-based routing)
- Install and configure TanStack Query (`query-client.ts`)
- Configure Tailwind CSS
- Set up DDD folder structure: `src/domains/`, `src/shared/`, `src/routes/`
- Implement `shared/lib/api-client.ts`:
  - In-memory token store (NOT localStorage)
  - Automatic `Authorization: Bearer <token>` header injection
  - 401 → clear token + redirect to `/login`
- Create root layout (`__root.tsx`) and placeholder `index.tsx` landing page
- Configure `VITE_API_URL` environment variable

### Docker

- `docker-compose.dev.yml`:
  - `postgres:16-alpine` service, port 5432, named volume
  - `valkey/valkey:latest` service, port 6379
  - `backend` service with hot-reload (`node ace serve --watch`)
  - `frontend` service with Vite dev server (`npm run dev`)
- `backend/Dockerfile.dev` and `backend/Dockerfile` (production multi-stage)
- `frontend/Dockerfile.dev` and `frontend/Dockerfile` (production Nginx static)
- `frontend/nginx.conf` with SPA fallback (`try_files $uri /index.html`), security headers, and asset caching

### Environment

- `backend/.env.example` with all required variables documented (see section 13 of PLAN.md)
- `frontend/.env.example` with `VITE_API_URL`

## Acceptance Criteria

- [ ] `docker compose -f docker-compose.dev.yml up` starts all four services without errors
- [ ] `GET http://localhost:3333/health` returns `200 { "status": "ok" }`
- [ ] `GET http://localhost:3000` serves the frontend landing page
- [ ] All migrations run cleanly (`node ace migration:run`)
- [ ] `node ace migration:rollback` and re-run succeeds (migrations are idempotent)
- [ ] All UUID primary keys are confirmed in DB schema (no auto-increment integer IDs)
- [ ] CORS rejects a request from an unlisted origin with `403`
- [ ] Global exception handler returns `{ error: { message: "..." } }` for unhandled errors — no stack traces in response
- [ ] Encrypted columns (`last_name`, `email`, `birth_date`, `weight_kg`, `height_cm`) store ciphertext in DB, not plaintext
- [ ] Frontend `api-client.ts` does not write the token to `localStorage` or `sessionStorage` at any point
- [ ] Frontend builds without TypeScript errors (`npm run build`)

## Tests

### Backend

- **Migration integrity:** After `migration:run`, assert all expected tables and columns exist with correct types (use `db.inspect()` or a raw query in a Japa test).
- **Health check:** `GET /health` → `200 { status: 'ok' }`.
- **CORS:** Request with `Origin: http://evil.com` header → response does not include `Access-Control-Allow-Origin`.
- **Encryption sanity:** Insert a row with an encrypted field via the model, then query the raw DB row and assert the stored value is not the plaintext original.
- **Exception handler:** Trigger a route that throws a generic `Error` and assert the response is `500` with a JSON body that does not contain a stack trace or file path.

### Frontend

- **Build check:** `npm run build` exits with code 0.
- **api-client unit test (Vitest):** After `setToken('abc')`, the next `apiClient()` call includes `Authorization: Bearer abc` in the request headers. After `clearToken()`, the header is absent.
- **401 redirect test (Vitest):** When the mock API returns 401, `apiClient` clears the token and navigates to `/login`.
