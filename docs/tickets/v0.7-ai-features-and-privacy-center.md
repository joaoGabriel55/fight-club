# v0.7 — AI Features & Privacy Center

## Main Goal

Add AI-powered student improvement tips via the Claude API (proxied through the backend to protect credentials and enforce privacy rules). Students can request martial arts training tips based on teacher feedback they've received — no free-text input allowed (dropdown-only UI to prevent prompt injection), with a mandatory disclaimer to always confirm tips with their Teacher/Sensei. Implement the privacy center: GDPR-style data export and full account erasure. After this ticket the app satisfies the AI bonus requirement and the privacy-by-design commitment.

---

## Scope

### Backend

#### AI — Student Improvement Tips

**Endpoint:** `POST /api/v1/ai/improvement-tips`

- Auth required. **Student only.**
- Request body: `{ feedback_id, focus_area? }` — only the feedback ID and an optional predefined focus area (no free text).
- `focus_area` is validated server-side against an allowed enum: `Striking technique`, `Ground game`, `Takedowns`, `Defense`, `Conditioning`, `Flexibility`, `Sparring strategy`, `Competition preparation`, `Forms / Kata`, `Footwork`.
- Backend resolves context from the feedback → enrollment → class: `feedback_content` (teacher-authored, decrypted), `martial_art`, `belt_name` (latest from `belt_progress`), `months_enrolled` (calculated from `enrollment.joined_at`).
- Constructs a prompt for Claude API using ONLY anonymized context — no student name, no email, no birth_date:

  ```
  System: You are a martial arts training advisor knowledgeable about: Kickboxing, Muay Thai, Brazilian Jiu-Jitsu (BJJ), Boxing, Wrestling (Olympic, Greco-Roman, Folkstyle), Catch Wrestling, Judo, Brazilian Submission Wrestling (Luta Livre), Karate, Capoeira, Taekwondo, Sanda/Sanshou, and Sambo. Provide specific, actionable improvement tips. Do not ask for personal information. Do not reference the student by name.

  User:
  Martial art: [martial_art]
  Current belt level: [belt_name or "No belt system for this art"]
  Time training: [months] month(s)
  Teacher's feedback: "[feedback_content]"
  Focus area: [focus_area or "General"]

  Based on this feedback, provide 3-5 specific, actionable improvement tips tailored to this student's level and martial art. Format as a numbered list.
  ```

- Calls Claude API (`claude-sonnet-4-6` model, `max_tokens: 500`).
- Returns `{ tips: "..." }`.
- Rate limited: 10 AI requests per student per minute (keyed by user ID).
- Policy check: student must own the enrollment that the feedback belongs to. Another student's feedback → `403`.

**Supported martial arts (initial list):**
Kickboxing, Muay Thai, Brazilian Jiu-Jitsu (BJJ), Boxing, Wrestling (Olympic, Greco-Roman, Folkstyle), Catch Wrestling, Judo, Brazilian Submission Wrestling (Luta Livre), Karate, Capoeira, Taekwondo, Sanda/Sanshou, Sambo.

**Environment variable:** `ANTHROPIC_API_KEY`

**Prompt injection prevention:** The student never types free text — the only user-controlled input is `focus_area`, which is enum-validated on the backend. The `feedback_content` is teacher-authored and already stored in the DB.

**Privacy rule enforced in backend:** AI prompts must never contain student PII. A unit test asserts this by intercepting the constructed prompt string and checking it does not contain the student's `first_name`, `last_name`, `email`, or `birth_date`.

#### Privacy Center

**Endpoints:**

`PrivacyController`:

- `GET /api/v1/privacy/my-data` — export all personal data as JSON. Returns:

  ```json
  {
    "account": { "first_name", "last_name", "email", "profile_type", "created_at" },
    "enrollments": [ { "class_name", "joined_at", "status" } ],
    "feedback_received": [ { "content", "created_at", "class_name" } ],
    "belt_progress": [ { "belt_name", "awarded_at", "class_name" } ],
    "announcements_received": [ { "title", "content", "class_name", "created_at" } ]
  }
  ```

  Encrypted fields are decrypted in this response (user is requesting their own data).

- `DELETE /api/v1/privacy/my-data` — full erasure (equivalent to `DELETE /api/v1/auth/me` but explicit privacy route). Triggers `UserAnonymizer.anonymize(user)`:
  1. Set `first_name = 'Deleted'`, `last_name = enc('User')`, `email = enc(sha256(original_email))`, `password_hash = ''`, `deleted_at = now()`
  2. Delete `student_profile` / `teacher_profile`
  3. Hard-delete all enrollments and cascade (feedback, belt_progress)
  4. Anonymize audit logs: set `user_id = null` (or a special "deleted" sentinel), retain the log entries for accountability.
  5. Return `204`.

- `GET /api/v1/privacy/policy` — return current privacy policy text as plain text or JSON `{ content: "..." }`. No auth required.

**`UserAnonymizer` service** (`app/services/user_anonymizer.ts`):

- Encapsulates all deletion/anonymization logic.
- Used by both `DELETE /auth/me` (v0.2) and `DELETE /privacy/my-data`.

**Audit logging**

- Log `data_export_requested`, `account_erasure_requested`.

### Frontend

**AI — Student Improvement Tips**

Update `FeedbackCard.tsx` (created in v0.5):

- Add "Get AI Tips" button in the card footer (visible only on student-facing pages).
- On click: open `ImprovementTipsDialog` modal.

New component `ImprovementTipsDialog.tsx`:

- Overlay modal (same pattern as `ConsentDialog`).
- Martial art displayed as read-only (pre-filled from class enrollment).
- **Focus area dropdown** (shadcn Select) with predefined options: Striking technique, Ground game, Takedowns, Defense, Conditioning, Flexibility, Sparring strategy, Competition preparation, Forms / Kata, Footwork. Optional — "General" is the default.
- **No free-text input anywhere** — prevents prompt injection. Student interacts only via buttons and the dropdown.
- "Get Tips" button → calls `POST /api/v1/ai/improvement-tips`.
- Loading spinner while waiting.
- On success: display tips (whitespace-pre-wrap).
- **Disclaimer banner** (always visible, above the tips):
  > "Always confirm these tips with your Teacher/Sensei and practice them with your teammates."
- On error: inline error message.
- "Close" button to dismiss.

```
  hooks/
    useImprovementTips.ts    # useMutation wrapping ai.service
  components/
    ImprovementTipsDialog.tsx  # Modal: focus area dropdown + "Get Tips" + disclaimer + tips display
```

**Privacy domain**

```
src/domains/privacy/
  services/
    privacy.service.ts       # exportMyData(), eraseMyData(), getPrivacyPolicy()
  hooks/
    useExportData.ts         # useMutation: triggers download of JSON blob
    useDeleteAccount.ts      # useMutation: calls erasure, then clearToken + redirect
  components/
    DataExportButton.tsx     # Button + loading state. On success: download JSON file as "my-fight-club-data.json".
    DeleteAccountDialog.tsx  # Confirmation modal: "This action cannot be undone." Requires typing "DELETE" to confirm. Calls useDeleteAccount.
    PrivacyPolicyViewer.tsx  # Renders the policy text from GET /privacy/policy.
```

**Route**

- `/privacy` — Privacy Center page with three sections:
  1. **My Data** — `DataExportButton`
  2. **Delete Account** — `DeleteAccountDialog` trigger
  3. **Privacy Policy** — `PrivacyPolicyViewer`

## Acceptance Criteria

- [ ] `POST /api/v1/ai/improvement-tips` returns a non-empty `tips` string. The Claude API prompt used internally does NOT contain any student name, email, or birth_date.
- [ ] `POST /api/v1/ai/improvement-tips` by a teacher → `403`.
- [ ] `POST /api/v1/ai/improvement-tips` with a `feedback_id` from another student's enrollment → `403`.
- [ ] `POST /api/v1/ai/improvement-tips` with an invalid `focus_area` value (not in the allowed enum) → `422`.
- [ ] AI endpoint returns `429` after 10 requests in 1 minute from the same student.
- [ ] `GET /api/v1/privacy/my-data` returns all personal data with encrypted fields decrypted. The response includes enrollments, feedback, belt progress.
- [ ] `GET /api/v1/privacy/my-data` by User A does not include any data belonging to User B.
- [ ] `DELETE /api/v1/privacy/my-data` → `204`. DB row has `deleted_at` set, PII anonymized, profile deleted, enrollments and cascades hard-deleted. Audit logs retained with anonymized user reference.
- [ ] `GET /api/v1/privacy/policy` returns `200` without authentication.
- [ ] Frontend "Get AI Tips" button appears on student feedback cards. Opens dialog with focus area dropdown (no free-text input).
- [ ] Frontend `ImprovementTipsDialog` shows loading spinner during request. On success, tips are displayed with disclaimer: "Always confirm these tips with your Teacher/Sensei and practice them with your teammates."
- [ ] Frontend `DataExportButton` triggers a file download of valid JSON.
- [ ] Frontend `DeleteAccountDialog` requires typing "DELETE" before the confirm button is enabled.
- [ ] After account deletion, the user is redirected to `/login` and the in-memory token is cleared.

## Tests

### Backend (Japa)

- **AI improvement tips — happy path:** Student requests tips for own feedback → 200 + `tips` string.
- **AI improvement tips — PII check:** Mock the Anthropic client. Assert the prompt string does not contain the student's `first_name`, `last_name`, `email`, or `birth_date`.
- **AI improvement tips — not student:** Teacher calls endpoint → `403`.
- **AI improvement tips — wrong student:** Student B tries Student A's feedback → `403`.
- **AI improvement tips — invalid focus_area:** Value not in allowed enum → `422`.
- **AI improvement tips — invalid feedback_id:** Random UUID → `404`.
- **AI improvement tips — rate limit:** 11 rapid requests → 11th returns `429`.
- **AI improvement tips — audit log:** After success, `audit_logs` contains `ai_tips_requested` entry.
- **Data export — own data only:** User A's export does not contain User B's feedback or belt data.
- **Data export — decrypted fields:** Exported `email` is the original plaintext email, not ciphertext.
- **Data export — completeness:** Export includes enrollments, feedback, belt_progress arrays.
- **Erasure — anonymization:** After `DELETE /privacy/my-data`, user row has `deleted_at` set, `first_name = 'Deleted'`, raw `email` column is not the original plaintext.
- **Erasure — cascade:** After erasure, the user's enrollments, feedback, and belt_progress are gone.
- **Erasure — audit log retention:** Audit log rows where `user_id` matched are retained but with user reference anonymized.
- **Privacy policy — no auth:** `GET /privacy/policy` returns `200` without a token.
- **Audit log:** `data_export_requested`, `account_erasure_requested` events logged.

### Frontend (Vitest)

- **DataExportButton:** On mutation success, `URL.createObjectURL` is called and a download link is clicked. On error, shows toast.
- **DeleteAccountDialog:** Confirm button disabled until user types "DELETE". Correct input enables button. On success, `clearToken` is called.
- **ImprovementTipsDialog:** Dialog renders with focus area dropdown (no free-text input). "Get Tips" click calls AI service. Loading spinner visible during request. On success, tips displayed with disclaimer banner. On error, inline error message shown.
- **FeedbackCard AI button:** "Get AI Tips" button visible when `showAiTips={true}`, hidden when `false`. Click opens `ImprovementTipsDialog`.

### E2E (Playwright)

- **AI improvement tips flow:** Student navigates to My Feedback → clicks "Get AI Tips" on a feedback card → dialog opens → selects focus area from dropdown → clicks "Get Tips" → spinner appears → tips displayed with disclaimer → closes dialog.
- **Privacy export:** Student navigates to `/privacy` → clicks "Export my data" → JSON file downloads → file contains the student's own data.
- **Account deletion:** Student → Privacy Center → Delete Account → types "DELETE" → confirms → redirected to `/login` → attempting to log in with same credentials fails (account anonymized).
