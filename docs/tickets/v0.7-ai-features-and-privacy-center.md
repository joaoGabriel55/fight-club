# v0.7 — AI Features & Privacy Center

## Main Goal

Add AI-powered feedback assistance and smart announcement drafting via the Claude API (proxied through the backend to protect credentials and enforce privacy rules). Implement the privacy center: GDPR-style data export and full account erasure. After this ticket the app satisfies the AI bonus requirement and the privacy-by-design commitment.

---

## Scope

### Backend

#### AI — Feedback Assistant

**Endpoint:** `POST /api/v1/ai/feedback-suggest`

- Auth required. Teacher only.
- Request body: `{ enrollment_id, draft_content? }` — only the enrollment ID (no PII in the request).
- Backend resolves context from the enrollment: `belt_name` (latest), `martial_art`, `months_enrolled` (calculated), optionally `draft_content` if teacher already started writing.
- Constructs a prompt for Claude API using ONLY anonymized context — no student name, no email, no birth_date:
  ```
  You are a martial arts coach assistant. The student has been practicing [martial_art] for [months] months and currently holds a [belt_name] belt.
  Help the teacher write constructive, encouraging feedback. Draft or improve this feedback:
  [draft_content or "No draft yet."]
  ```
- Calls Claude API (`claude-sonnet-4-6` model, `max_tokens: 500`).
- Returns `{ suggestion: "..." }`.
- Rate limited: 10 AI requests per teacher per minute.

**Endpoint:** `POST /api/v1/ai/announcement-draft`

- Auth required. Teacher only.
- Request body: `{ class_id, prompt }` — teacher's natural language description.
- Backend verifies teacher owns the class.
- Constructs Claude prompt: "Draft a clear, formatted class announcement based on this description: [prompt]"
- Returns `{ draft: "..." }`.
- Rate limited: 10 AI requests per teacher per minute.

**Environment variable:** `ANTHROPIC_API_KEY`

**Privacy rule enforced in backend:** AI prompts must never contain student PII. A unit test asserts this by inspecting the constructed prompt string.

#### Privacy Center

**Endpoints:**

`PrivacyController`:

- `GET /api/v1/privacy/my-data` — export all personal data as JSON. Returns:
  ```json
  {
    "account": { "first_name", "last_name", "email", "profile_type", "created_at" },
    "enrollments": [ { "class_name", "joined_at", "status" } ],
    "feedback_received": [ { "content", "created_at", "class_name" } ],
    "belt_progress": [ { "belt_name", "awarded_at", "class_name" } ],
    "announcements_received": [ { "title", "content", "class_name", "created_at" } ]
  }
  ```
  Encrypted fields are decrypted in this response (user is requesting their own data).

- `DELETE /api/v1/privacy/my-data` — full erasure (equivalent to `DELETE /api/v1/auth/me` but explicit privacy route). Triggers `UserAnonymizer.anonymize(user)`:
  1. Set `first_name = 'Deleted'`, `last_name = enc('User')`, `email = enc(sha256(original_email))`, `password_hash = ''`, `deleted_at = now()`
  2. Delete `student_profile` / `teacher_profile`
  3. Hard-delete all enrollments and cascade (feedback, belt_progress)
  4. Anonymize audit logs: set `user_id = null` (or a special "deleted" sentinel), retain the log entries for accountability.
  5. Return `204`.

- `GET /api/v1/privacy/policy` — return current privacy policy text as plain text or JSON `{ content: "..." }`. No auth required.

**`UserAnonymizer` service** (`app/services/user_anonymizer.ts`):
- Encapsulates all deletion/anonymization logic.
- Used by both `DELETE /auth/me` (v0.2) and `DELETE /privacy/my-data`.

**Audit logging**
- Log `data_export_requested`, `account_erasure_requested`.

## Acceptance Criteria

- [ ] `POST /api/v1/ai/feedback-suggest` returns a non-empty `suggestion` string. The Claude API prompt used internally does NOT contain any student name, email, or birth_date.
- [ ] `POST /api/v1/ai/feedback-suggest` by a student → `403`.
- [ ] `POST /api/v1/ai/feedback-suggest` with an `enrollment_id` from a class the teacher does not own → `403`.
- [ ] `POST /api/v1/ai/announcement-draft` returns a non-empty `draft` string.
- [ ] `POST /api/v1/ai/announcement-draft` with a `class_id` the teacher does not own → `403`.
- [ ] AI endpoints return `429` after 10 requests in 1 minute from the same teacher.
- [ ] `GET /api/v1/privacy/my-data` returns all personal data with encrypted fields decrypted. The response includes enrollments, feedback, belt progress.
- [ ] `GET /api/v1/privacy/my-data` by User A does not include any data belonging to User B.
- [ ] `DELETE /api/v1/privacy/my-data` → `204`. DB row has `deleted_at` set, PII anonymized, profile deleted, enrollments and cascades hard-deleted. Audit logs retained with anonymized user reference.
- [ ] `GET /api/v1/privacy/policy` returns `200` without authentication.
- [ ] Frontend "Suggest with AI" button is disabled during loading. On success, textarea is populated.
- [ ] Frontend `DataExportButton` triggers a file download of valid JSON.
- [ ] Frontend `DeleteAccountDialog` requires typing "DELETE" before the confirm button is enabled.
- [ ] After account deletion, the user is redirected to `/login` and the in-memory token is cleared.

## Tests

### Backend (Japa)

- **AI feedback suggest — PII check:** Intercept the Claude API call (mock the HTTP client). Assert the prompt string does not contain the student's `first_name`, `last_name`, `email`, or `birth_date`.
- **AI feedback suggest — not teacher:** → `403`.
- **AI feedback suggest — not class owner:** → `403`.
- **AI feedback suggest — rate limit:** 11 rapid requests → 11th returns `429`.
- **AI announcement draft — not owner:** → `403`.
- **Data export — own data only:** User A's export does not contain User B's feedback or belt data.
- **Data export — decrypted fields:** Exported `email` is the original plaintext email, not ciphertext.
- **Data export — completeness:** Export includes enrollments, feedback, belt_progress arrays.
- **Erasure — anonymization:** After `DELETE /privacy/my-data`, user row has `deleted_at` set, `first_name = 'Deleted'`, raw `email` column is not the original plaintext.
- **Erasure — cascade:** After erasure, the user's enrollments, feedback, and belt_progress are gone.
- **Erasure — audit log retention:** Audit log rows where `user_id` matched are retained but with user reference anonymized.
- **Privacy policy — no auth:** `GET /privacy/policy` returns `200` without a token.
- **Audit log:** `data_export_requested`, `account_erasure_requested` events logged.

### Frontend (Vitest)

- **DataExportButton:** On mutation success, `URL.createObjectURL` is called and a download link is clicked. On error, shows toast.
- **DeleteAccountDialog:** Confirm button disabled until user types "DELETE". Correct input enables button. On success, `clearToken` is called.
- **FeedbackForm AI button:** "Suggest with AI" click calls the AI service. Loading spinner visible during request. On success, textarea value is updated.

### E2E (Playwright)

- **AI feedback flow:** Teacher sends feedback → clicks "Suggest with AI" → spinner appears → suggestion appears in textarea → teacher edits and submits.
- **Privacy export:** Student navigates to `/privacy` → clicks "Export my data" → JSON file downloads → file contains the student's own data.
- **Account deletion:** Student → Privacy Center → Delete Account → types "DELETE" → confirms → redirected to `/login` → attempting to log in with same credentials fails (account anonymized).
