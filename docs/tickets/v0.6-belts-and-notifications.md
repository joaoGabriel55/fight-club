# v0.6 — Belt Progression & Notifications

## Main Goal

Teachers can award belts to students in classes that have the belt system enabled. Students see a full timeline of their belt history. The notification system (started in v0.5) is completed: students and teachers can view their notifications, mark them as read, and notifications automatically expire after 90 days. After this ticket the core product is feature-complete.

---

## Scope

### Backend

**Migrations (new)**

- `belt_progress` — `id` (UUID), `enrollment_id` (FK → enrollments), `belt_name` (varchar), `awarded_at` (timestamp), `awarded_by` (FK → users)
- `notifications` table was defined in v0.5. No new migration needed if created there; otherwise create it here:
  - `id` (UUID), `user_id` (FK → users), `type` (varchar), `title`, `body`, `read_at` (nullable timestamp), `data` (JSON), `created_at`, `expires_at`

**Models**

- `BeltProgress` model: `belongsTo` Enrollment, `belongsTo` User (awarder)
- `Notification` model: `belongsTo` User

**Bouncer policy — `BeltPolicy`**

- `award(user, enrollment)` — user is the teacher of the enrollment's class AND the class has `has_belt_system = true`
- `view(user, enrollment)` — user is the teacher OR the enrolled student

**Bouncer policy — `NotificationPolicy`**

- `view(user, notification)` — notification.user_id === user.id
- `markRead(user, notification)` — notification.user_id === user.id

**VineJS validators**

- `AwardBeltValidator`: `belt_name` (min 1, max 100), `awarded_at` (valid date, not in future)

**Controllers**

`BeltProgressController`:

- `POST /api/v1/enrollments/:enrollmentId/belts` — award belt:
  1. Verify teacher owns the class (Bouncer: `BeltPolicy.award`)
  2. Verify class has `has_belt_system = true` — if not, return `422` with message "This class does not use a belt system"
  3. Create `BeltProgress` row
  4. Emit `BeltAwarded` event
  5. Return `201` with belt data
- `GET /api/v1/enrollments/:enrollmentId/belts` — belt history, sorted by `awarded_at` asc (Bouncer: `BeltPolicy.view`)

`NotificationsController`:

- `GET /api/v1/notifications` — list authenticated user's notifications, excluding expired (`expires_at < now()`), sorted by `created_at` desc, with unread count in response metadata
- `PUT /api/v1/notifications/:id/read` — mark single notification as read (Bouncer: `NotificationPolicy.markRead`)
- `PUT /api/v1/notifications/read-all` — mark all of the authenticated user's unread notifications as read. Return `{ updated: N }`.

**Events & Listeners**

- `BeltAwarded` → listener `CreateNotification`: create notification for the student with `type = 'belt_awarded'`, title like "New belt awarded!", body with belt name and class name.

**`NotificationService`**

- `createNotification(userId, type, title, body, data, ttlDays = 90)` — shared service used by all event listeners. Sets `expires_at = now() + ttlDays days`.
- Used by: `FeedbackSent`, `AnnouncementCreated`, `BeltAwarded`, `StudentEnrolled` listeners.

**Audit logging**

- Log `belt_awarded` event.

**Factories**

- `BeltProgressFactory`
- `NotificationFactory`

## Acceptance Criteria

- [ ] `POST /api/v1/enrollments/:enrollmentId/belts` by the class teacher when `has_belt_system = true` → `201`.
- [ ] `POST /api/v1/enrollments/:enrollmentId/belts` when `has_belt_system = false` → `422` with message about belt system being disabled.
- [ ] `POST /api/v1/enrollments/:enrollmentId/belts` by a non-teacher → `403`.
- [ ] `POST /api/v1/enrollments/:enrollmentId/belts` by the student themselves → `403`.
- [ ] `GET /api/v1/enrollments/:enrollmentId/belts` returns belts in chronological order.
- [ ] `GET /api/v1/enrollments/:enrollmentId/belts` by a different student (not the enrolled one) → `403`.
- [ ] `BeltAwarded` event creates a notification for the student with type `belt_awarded`.
- [ ] `GET /api/v1/notifications` does not return expired notifications (`expires_at < now()`).
- [ ] `PUT /api/v1/notifications/:id/read` by a different user → `403`.
- [ ] `PUT /api/v1/notifications/read-all` marks only the authenticated user's notifications, not all users'.
- [ ] `GET /api/v1/notifications` response includes `{ data: [...], meta: { unread_count: N } }`.
- [ ] Frontend `NotificationBell` shows the correct unread count badge.
- [ ] `BeltTimeline` shows all awarded belts in chronological order.
- [ ] `AwardBeltForm` is hidden when the class has `has_belt_system = false`.
- [ ] Clicking a notification in the panel marks it as read and updates the unread count.

## Tests

### Backend (Japa)

- **Award belt — happy path:** Teacher awards belt to enrolled student in belt-enabled class → `201`, row in DB.
- **Award belt — no belt system:** `has_belt_system = false` → `422`.
- **Award belt — not teacher:** Student → `403`. Non-owner teacher → `403`.
- **Belt history — isolation:** Student A cannot view Student B's belt history → `403`.
- **Belt history — order:** Multiple belts awarded at different times → returned in ascending `awarded_at` order.
- **Belt awarded notification:** After award, student has notification with `type = 'belt_awarded'`.
- **Notification expiry:** Insert a notification with `expires_at` in the past. `GET /notifications` does not return it.
- **Notification mark read:** `PUT /notifications/:id/read` → `read_at` set. `GET /notifications` shows `read_at` for that item.
- **Mark all read:** 3 unread notifications → `PUT /notifications/read-all` → `{ updated: 3 }`. All now have `read_at` set.
- **Mark read — not owner:** → `403`.
- **Unread count:** `GET /notifications` meta includes correct `unread_count`.
- **Audit log:** `belt_awarded` event logged.

### Frontend (Vitest)

- **BeltTimeline:** Renders all belts in order. Empty state shows "No belts awarded yet".
- **AwardBeltForm:** Hidden when `class.has_belt_system = false`. Visible when true. Validates belt_name not empty.
- **NotificationBell:** Shows badge with count when `unread_count > 0`. No badge when 0.
- **NotificationItem:** Has unread visual style when `read_at === null`. Clicking fires `useMarkRead` mutation.
- **useNotifications:** Configured with `refetchInterval: 60000`.

### E2E (Playwright)

- **Belt flow:** Teacher logs in → opens class (belt system enabled) → selects student → awards belt "Blue Belt" → student logs in → navigates to progress → sees "Blue Belt" in timeline → sees notification in bell.
- **No belt system:** Teacher opens class without belt system → student profile view does not show `AwardBeltForm`.
- **Notification mark all read:** Student has 3 unread notifications → clicks "Mark all read" → badge disappears.
