# v0.5 — Announcements & Feedback

## Main Goal

Teachers can post announcements to their classes and send private feedback to individual students. Students can read announcements for classes they're enrolled in and view their own feedback history. Feedback content is encrypted at rest. After this ticket the core communication loop is working.

---

## Scope

### Backend

**Migrations (new)**
- `announcements` — `id` (UUID), `class_id` (FK → classes), `author_id` (FK → users), `title` (varchar), `content` (text), `created_at`
- `feedback` — `id` (UUID), `enrollment_id` (FK → enrollments), `teacher_id` (FK → users), `content` (text, encrypted), `created_at`

**Models**
- `Announcement` model: `belongsTo` Class, `belongsTo` User (author)
- `Feedback` model: `belongsTo` Enrollment, `belongsTo` User (teacher). `content` column uses AdonisJS encryption (`@column({ serialize: ... })`).

**Bouncer policy — `AnnouncementPolicy`**
- `create(user, class)` — user is teacher of that class
- `delete(user, announcement)` — user is teacher of announcement's class
- `view(user, class)` — user is teacher of the class OR user is an active student in the class

**Bouncer policy — `FeedbackPolicy`**
- `create(user, enrollment)` — user is the teacher of the enrollment's class
- `view(user, enrollment)` — user is the teacher of the enrollment's class OR user is the enrolled student (enrollment.student_id === user.id)

**VineJS validators**
- `CreateAnnouncementValidator`: `title` (min 3, max 255), `content` (min 10)
- `CreateFeedbackValidator`: `content` (min 10)

**Controllers**

`AnnouncementsController`:
- `POST /api/v1/classes/:classId/announcements` — create announcement (teacher must own class). Emit `AnnouncementCreated` event.
- `GET /api/v1/classes/:classId/announcements` — list announcements, newest first. Requires user to be either the teacher or an enrolled student of the class.
- `GET /api/v1/announcements` — student view: all announcements across all of the student's active enrollments, sorted by `created_at` desc. Teacher calling this → `403`.
- `DELETE /api/v1/classes/:classId/announcements/:id` — delete announcement (teacher must own class).

`FeedbackController`:
- `POST /api/v1/enrollments/:enrollmentId/feedback` — teacher sends feedback. Validates teacher owns the enrollment's class. Emit `FeedbackSent` event.
- `GET /api/v1/enrollments/:enrollmentId/feedback` — list feedback for enrollment. Teacher or the enrolled student.
- `GET /api/v1/feedback` — student view: all feedback across all of the student's enrollments. Teacher calling this → `403`.

**Events & Listeners**
- `AnnouncementCreated` → listener `CreateNotification`: create a notification for every active student in the class.
- `FeedbackSent` → listener `CreateNotification`: create a notification for the student who received the feedback.

**Notifications model (new)**
```
notifications — id (UUID), user_id (FK), type (varchar), title, body, read_at (nullable), data (JSON), created_at, expires_at
```

**Serialization**
- Feedback `content` is decrypted on read — but only when the requester passes the Bouncer check.
- Announcement list does not expose `author_id` raw — return `{ author: { first_name } }`.

**Audit logging**
- Log `announcement_created`, `announcement_deleted`, `feedback_sent`, `feedback_viewed` events.

### Frontend

**Announcements domain**
```
src/domains/announcements/
  schemas/
    announcement.schema.ts   # Zod: title (min 3, max 255), content (min 10)
  services/
    announcements.service.ts # getClassAnnouncements(), getMyAnnouncements(), createAnnouncement(), deleteAnnouncement()
  hooks/
    useAnnouncements.ts      # useQuery(['announcements', classId])
    useMyAnnouncements.ts    # useQuery(['my-announcements'])
    useCreateAnnouncement.ts # useMutation, invalidates class announcements
    useDeleteAnnouncement.ts # useMutation
  types/
    announcement.types.ts
  components/
    AnnouncementCard.tsx     # Card with title, content (truncated), author, date. Delete button for teacher.
    AnnouncementForm.tsx     # RHF + Zod form for creating announcements.
    AnnouncementFeed.tsx     # List of AnnouncementCard + empty state + create button (teacher only).
```

**Feedback domain**
```
src/domains/feedback/
  schemas/
    feedback.schema.ts       # Zod: content (min 10)
  services/
    feedback.service.ts      # getEnrollmentFeedback(), getMyFeedback(), sendFeedback()
  hooks/
    useFeedback.ts           # useQuery(['feedback', enrollmentId])
    useMyFeedback.ts         # useQuery(['my-feedback'])
    useSendFeedback.ts       # useMutation
  types/
    feedback.types.ts
  components/
    FeedbackCard.tsx         # Card with content, teacher name, date.
    FeedbackForm.tsx         # Textarea + submit. Used by teacher on student profile view. (AI button placeholder added here for v0.7.)
    FeedbackHistory.tsx      # List of FeedbackCard.
```

**Routes**
- `/classes/$classId/announcements` — `AnnouncementFeed` (teacher view: create + list + delete; student view: list only)
- `/feedback` — student route: `FeedbackHistory` with all feedback across enrollments
- Within class detail (teacher): student profile sub-view showing feedback history + `FeedbackForm`

## Acceptance Criteria

- [ ] `POST /api/v1/classes/:classId/announcements` by the class owner → `201`. Non-owner teacher → `403`. Student → `403`.
- [ ] `GET /api/v1/classes/:classId/announcements` by an enrolled student → `200` with announcements. By a non-enrolled user → `403`.
- [ ] `GET /api/v1/announcements` (student) returns announcements from ALL of the student's enrolled classes, not just one.
- [ ] `GET /api/v1/announcements` by a teacher → `403`.
- [ ] `DELETE /api/v1/classes/:classId/announcements/:id` by non-owner → `403`.
- [ ] `POST /api/v1/enrollments/:enrollmentId/feedback` by the class teacher → `201`. By a non-teacher → `403`. By the student themselves → `403`.
- [ ] `GET /api/v1/enrollments/:enrollmentId/feedback` by the enrolled student → `200`. By another student → `403`.
- [ ] `GET /api/v1/feedback` by a teacher → `403`.
- [ ] Feedback `content` is stored encrypted in DB (raw DB value is not the plaintext). Decrypted value is returned in the API response.
- [ ] `AnnouncementCreated` event creates notifications for all enrolled students of the class.
- [ ] `FeedbackSent` event creates a notification for the recipient student.
- [ ] Frontend announcement form: empty title → inline validation error. Cannot submit.
- [ ] Teacher sees "Delete" button on announcement cards; student does not.

## Tests

### Backend (Japa)

- **Create announcement — happy path:** `POST /announcements` by owner → `201`, row in DB.
- **Create announcement — non-owner teacher:** → `403`.
- **Create announcement — student:** → `403`.
- **List announcements — enrolled student:** `GET /classes/:id/announcements` → `200` with correct data.
- **List announcements — non-enrolled user:** → `403`.
- **My announcements — cross-class:** Student enrolled in 2 classes, each with 1 announcement. `GET /announcements` returns 2 items.
- **My announcements — teacher:** → `403`.
- **Delete announcement — non-owner:** → `403`.
- **Send feedback — happy path:** `POST /enrollments/:id/feedback` by class teacher → `201`.
- **Send feedback — not teacher:** Student tries to send feedback to themselves → `403`.
- **Feedback content encrypted:** After inserting feedback, query raw DB row — `content` column value differs from submitted plaintext.
- **Feedback list — data isolation:** Student A cannot see feedback sent to Student B in the same class (different enrollments).
- **My feedback — teacher:** `GET /feedback` → `403`.
- **AnnouncementCreated notifications:** After creating announcement, each enrolled student has a new row in `notifications`.
- **FeedbackSent notification:** After sending feedback, the student has a notification with `type = 'feedback_received'`.
- **Audit log:** `announcement_created`, `feedback_sent` events logged.

### Frontend (Vitest)

- **AnnouncementCard:** Shows title and truncated content. Teacher role shows Delete button; student role does not.
- **AnnouncementForm:** Empty title → error. Content too short → error. Submit calls mutation with correct payload.
- **FeedbackForm:** Empty content → error. Submit calls `useSendFeedback` mutation.
- **useSendFeedback:** On success, invalidates `['feedback', enrollmentId]` query.
- **AnnouncementFeed:** Empty state shows "No announcements yet" message.

### E2E (Playwright)

- **Announcement flow:** Teacher logs in → navigates to class → announcements tab → creates announcement → announcement appears in feed. Student logs in → navigates to same class → sees the announcement (but no create/delete controls).
- **Feedback flow:** Teacher navigates to student's profile within class → writes feedback → sends → notification appears for student → student logs in → sees feedback in `/feedback`.
- **Data isolation:** Student B cannot see Student A's feedback by navigating to enrollment URL.
