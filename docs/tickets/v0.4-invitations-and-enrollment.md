# v0.4 — Invitations & Enrollment

## Main Goal

Teachers generate shareable invite links. Students open those links, review a consent dialog explaining what data will be shared, accept, and join the class. Students can also view their enrollments and leave a class (which deletes their enrollment data). After this ticket the core teacher-student connection loop is functional end-to-end.

---

## Scope

### Backend

**Migrations (new)**

- `invitations` — `id` (UUID), `class_id` (FK → classes), `token` (UUID, unique), `expires_at`, `is_active` (bool, default true), `max_uses` (int, nullable), `use_count` (int, default 0), `created_at`
- `enrollments` — `id` (UUID), `class_id` (FK → classes), `student_id` (FK → users), `status` (`active` | `left`), `joined_at`, `left_at` (nullable), `data_consent_at`, unique constraint on `(class_id, student_id)`

**Models**

- `Invitation` model: `belongsTo` Class
- `Enrollment` model: `belongsTo` Class, `belongsTo` User (student), `hasMany` BeltProgress (stub), `hasMany` Feedback (stub)

**Bouncer policy — `InvitationPolicy`**

- `create(user, class)` — user is teacher of that class
- `list(user, class)` — user is teacher of that class
- `revoke(user, invitation)` — user is teacher of the invitation's class

**Bouncer policy — `EnrollmentPolicy`**

- `leave(user, enrollment)` — user.id === enrollment.student_id

**VineJS validators**

- `CreateInvitationValidator`: `expires_at` (ISO date, must be future), `max_uses` (positive int, nullable)
- `JoinClassValidator`: `consent` (must be `true`)

**Controllers**

`InvitationsController` (routes under `/api/v1/classes/:classId/invitations`, auth middleware):

- `POST /` — generate invitation: create row with UUID token, set expiry + max_uses. Return `{ id, token, invite_url, expires_at, max_uses }`. The `invite_url` is constructed as `${APP_URL}/join/${token}` — the URL does not contain the class name or any PII.
- `GET /` — list active (non-expired, is_active) invitations for the class (Bouncer: must own class)
- `DELETE /api/v1/invitations/:id` — revoke invitation (set `is_active = false`) (Bouncer: must own invitation's class)

`EnrollmentController`:

- `POST /api/v1/join/:token` — join class via token:
  1. Look up invitation by token
  2. Validate: exists, is_active, not expired, use_count < max_uses (if set)
  3. Validate: student not already enrolled in this class
  4. Require `consent: true` in body
  5. Create enrollment row with `data_consent_at = now()`
  6. Increment `invitation.use_count`
  7. Return `201` with enrollment data
  8. Emit `StudentEnrolled` event
- `GET /api/v1/enrollments` — list authenticated student's own enrollments (status `active`), with class name + schedule + teacher's first name
- `DELETE /api/v1/enrollments/:id` — leave class:
  1. Verify enrollment belongs to the authenticated student (Bouncer)
  2. Hard delete: remove enrollment row, delete all feedback received in this enrollment, delete belt progress for this enrollment
  3. Return `204`

**Events**

- `StudentEnrolled` event → listener `CreateNotification`: notify the teacher that a new student joined.

**Audit logging**

- Log `student_enrolled`, `student_left`, `invitation_created`, `invitation_revoked` events.

**Token cleanup (stub — scheduler wired in v0.8)**

- `CleanupExpiredInvitationsJob` — defined but not yet scheduled. Can be run manually via `node ace scheduler:run` or test helper.

**Routes**

- `/join/$token` — public route (no auth required before this page):
  1. If user is not authenticated: show "You need an account to join. Register or login." with links.
  2. If authenticated as student: show class name (fetched via token), then `ConsentDialog`. On accept → call `joinClass`.
  3. If authenticated as teacher: show "Teachers cannot join classes as students."
  4. If token is expired/invalid: show clear error message.
- `/classes/$classId/invitations` — `InvitationManager` (teacher-only, inside class detail tabs)
- `/enrollments` or `/dashboard` student section — list of enrolled classes

## Acceptance Criteria

- [ ] `POST /api/v1/classes/:classId/invitations` by the class owner → `201` with a token and `invite_url`. The URL contains only the UUID token — no class name, teacher name, or other identifying info.
- [ ] `POST /api/v1/join/:token` with `consent: true` → `201` enrollment. The enrollment row has `data_consent_at` set.
- [ ] `POST /api/v1/join/:token` without `consent: true` → `422`.
- [ ] `POST /api/v1/join/:token` with an expired token → `410 Gone`.
- [ ] `POST /api/v1/join/:token` with a revoked token (`is_active = false`) → `410 Gone`.
- [ ] `POST /api/v1/join/:token` when student is already enrolled in the same class → `409 Conflict`.
- [ ] `POST /api/v1/join/:token` when `use_count >= max_uses` → `410 Gone`.
- [ ] `GET /api/v1/enrollments` returns only the authenticated student's own enrollments — not other students'.
- [ ] `DELETE /api/v1/enrollments/:id` by a student who does NOT own the enrollment → `403`.
- [ ] After leaving a class, all feedback and belt progress for that enrollment are hard-deleted from the DB.
- [ ] The `StudentEnrolled` event creates a notification for the teacher.
- [ ] Frontend `/join/$token` page shows the `ConsentDialog` before enrollment. The Confirm button is disabled until the checkbox is checked.
- [ ] Copying the invite link from `InvitationManager` puts the correct URL in the clipboard.

## Tests

### Backend (Japa)

- **Generate invitation — happy path:** Teacher creates invitation → token is a UUID, `invite_url` contains only the token, no PII.
- **Generate invitation — not owner:** Non-owner teacher → `403`.
- **Join — happy path:** Valid token, consent given → `201`, enrollment in DB with `data_consent_at`, `invitation.use_count` incremented by 1.
- **Join — no consent:** Body without `consent: true` → `422`.
- **Join — expired token:** Token with `expires_at` in the past → `410`.
- **Join — revoked token:** `is_active = false` → `410`.
- **Join — max uses exhausted:** `use_count >= max_uses` → `410`.
- **Join — duplicate enrollment:** Same student, same class → `409`.
- **Join — teacher trying to join:** Token for a class → teacher account calls join → `403` (teachers cannot enroll as students).
- **Enrollments list — isolation:** Student A and Student B both enrolled in same class. Student A's `GET /enrollments` returns only Student A's enrollment.
- **Leave — cascade delete:** After `DELETE /enrollments/:id`, enrollment row gone, associated feedback gone, associated belt_progress gone.
- **Leave — not owner:** Student B cannot delete Student A's enrollment → `403`.
- **Notification created:** After join, teacher has a notification in DB.
- **Audit log:** `student_enrolled`, `student_left`, `invitation_created` events logged.

### Frontend (Vitest)

- **ConsentDialog:** Confirm button is disabled when checkbox is unchecked. Enabled after checking. Clicking cancel does not call `joinClass`.
- **InviteLinkCard:** "Copy link" button calls `navigator.clipboard.writeText` with the correct URL.
- **useJoinClass:** On success, navigates to `/dashboard`. On `409` error, shows "already enrolled" message.

### E2E (Playwright)

- **Full join flow:** Teacher logs in → creates class → navigates to invitations tab → generates invite link → copies link → logs out → student logs in → visits invite URL → sees consent dialog → checks consent → confirms → lands on dashboard with class listed in enrollments.
- **Expired token UX:** Visit `/join/<expired-token>` → see a clear "This invitation has expired" message.
- **Leave class:** Student enrolled → navigates to enrollment → clicks "Leave class" → confirms → class disappears from list.
