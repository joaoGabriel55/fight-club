# v1.0 — Deploy & Documentation

## Main Goal

Ship the app to production: CI/CD pipeline, Docker production builds, deployment to a live host, seed data for the demo, HTTPS enforced, and a complete README. After this ticket the app is live, documented, and ready for submission.

---

## Scope

### CI/CD — GitHub Actions

**`.github/workflows/ci.yml`** — runs on every push to `main` and on PRs:

```yaml
jobs:
  backend-test:
    - Checkout
    - Setup Node 20
    - Install deps (npm ci)
    - Start PostgreSQL service container (postgres:16-alpine)
    - Run migrations (node ace migration:run)
    - Run Japa tests (node ace test)

  frontend-test:
    - Checkout
    - Setup Node 20
    - Install deps (npm ci)
    - Run Vitest (npm run test)
    - Run build check (npm run build)

  e2e:
    - Checkout
    - Setup Node 20
    - Start full stack via docker-compose
    - Wait for health check
    - Run Playwright tests (npm run test:e2e)
    - Upload Playwright report as artifact
```

**`.github/workflows/deploy.yml`** — runs on push to `main` after CI passes:

- Build and push Docker images to registry (GitHub Container Registry or Docker Hub)
- SSH into production server and pull new images
- Run `docker compose -f docker-compose.prod.yml up -d`
- Run `node ace migration:run` against production DB
- Health check: `curl https://api.fightclub.app/health` → 200

### Production Deployment

**Platform:** Render (or self-hosted VPS with Docker) — document which one is used.

**Backend service:**

- Docker container from `backend/Dockerfile`
- Environment variables set via Render dashboard or `.env` file on VPS
- PostgreSQL: Render managed DB or self-hosted
- Redis: Render managed Redis or self-hosted
- Domain: `api.fightclub.app` (or equivalent) with HTTPS enforced

**Frontend service:**

- Built static files served by Nginx via `frontend/Dockerfile`
- Domain: `fightclub.app` with HTTPS enforced
- Nginx config from `frontend/nginx.conf` with SPA fallback

**HTTPS:**

- TLS certificate via Let's Encrypt (Certbot) or Render's managed TLS
- HTTP → HTTPS redirect enforced at Nginx/Render level
- `Strict-Transport-Security` header set (already handled by backend middleware from v0.8)

### Production Seeding

**`node ace db:seed --files database/seeders/demo_seeder.ts`**

Demo seeder creates:

- 1 teacher user: `teacher@demo.com` / `Demo1234!`
- 1 student user: `student@demo.com` / `Demo1234!`
- 2 classes (one with belt system, one without)
- Schedules for each class
- 1 active invitation per class
- Student enrolled in both classes (with consent)
- 3 announcements across classes
- 2 feedback entries for the student
- 3 belt progress entries (for the belt-system class)
- 5 notifications (mix of read/unread)

Seeder is idempotent: running it twice does not duplicate data (check-before-insert pattern).

## Tests

### Infrastructure / Smoke Tests

- **Health check:** `GET /health` → `200`. Automated in deploy pipeline.
- **HTTPS redirect:** `curl -I http://api.fightclub.app/health` → `301` or `308` with `Location: https://...`
- **Security headers on prod:** `curl -I https://api.fightclub.app/health` returns `X-Content-Type-Options`, `X-Frame-Options`, `Strict-Transport-Security`.
- **CORS on prod:** Request from an unlisted origin → no `Access-Control-Allow-Origin` header in response.
- **Demo credentials work:** Playwright smoke test:
  1. Navigate to live URL
  2. Log in as `teacher@demo.com`
  3. Assert dashboard renders with at least 1 class card
  4. Log out → log in as `student@demo.com`
  5. Assert dashboard renders with at least 1 enrollment
- **No secrets in repo:** `git log --all -S 'ANTHROPIC_API_KEY' -- .` returns no commits (CI step).

### CI Verification

- All jobs in `ci.yml` pass on a fresh clone with no cached state.
- `deploy.yml` correctly updates the live service (verify by checking a version header or `/health` response after deploy).
